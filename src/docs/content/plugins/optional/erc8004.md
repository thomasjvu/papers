# ERC-8004 Quick Start Guide

Get your AI agent registered on the ERC-8004 "Trustless Agents" standard in 5 minutes.

## Prerequisites

- Phantasy Agent installed and running
- Wallet with testnet funds (Base Sepolia)
  - Get testnet ETH: [Base Sepolia Faucet](https://www.coinbase.com/faucets/base-ethereum-goerli-faucet)
- Private key for transactions

## Step 1: Get Testnet Funds

1. Visit [Base Sepolia Faucet](https://www.coinbase.com/faucets/base-ethereum-goerli-faucet)
2. Enter your wallet address
3. Receive free testnet ETH (~0.1 ETH)

## Step 2: Configure Agent

Create or update `config/agents/my-agent.json`:

```json
{
  "id": "my-agent",
  "name": "My Agent",
  "bio": "My AI assistant with ERC-8004 integration",

  "erc8004Config": {
    "enabled": true,
    "networks": ["base-sepolia"],
    "trustModels": ["reputation"],
    "autoSync": true
  },

  "plugins": ["erc8004"]
}
```

## Step 3: Set Environment Variables

```bash
# Set your wallet private key
export WALLET_PRIVATE_KEY=0x...

# Or use Coinbase wallet
export COINBASE_PRIVATE_KEY=0x...
```

## Step 4: Start Agent

```bash
bun run dev
```

## Step 5: Enable Plugin

Visit admin UI at `http://localhost:2000`:

1. Navigate to **Plugins**
2. Find "ERC-8004 Trustless Agents"
3. Click **Enable**

## Step 6: Register Your Agent

### Option A: Via Chat

Talk to your agent:

```
Please register me on the Base Sepolia ERC-8004 identity registry.
```

The agent will use the `register_agent_erc8004` tool and respond with:

```
Success! I've registered you as agent ID 123 on Base Sepolia.
Identity Registry: 0x...
Transaction: 0xabc...
```

### Option B: Via Admin API

```bash
curl -X POST http://localhost:2000/admin/api/plugins/erc8004/register \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"network": "base-sepolia"}'
```

## Step 7: View Your Agent Metadata

Visit `http://localhost:2000/agent.json` to see your ERC-8004 compliant metadata:

```json
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "My Agent",
  "description": "My AI assistant with ERC-8004 integration",
  "registrations": [
    {
      "chainId": 84532,
      "registry": "0x...",
      "agentId": "123"
    }
  ],
  "supportedTrust": ["reputation"],
  "capabilities": ["chat"]
}
```

## Step 8: Build Reputation

### Request Validation

Request third-party validation through the plugin-scoped admin surface:

```bash
curl -X POST http://localhost:2000/admin/api/plugins/erc8004/validation/request \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "agentId": "123",
    "validatorAddress": "0xValidator...",
    "requestUri": "ipfs://your-validation-request",
    "requestHash": "0xYourRequestHash",
    "network": "base-sepolia"
  }'
```

### Check Reputation

Ask your agent:

```
What's my current reputation on Base Sepolia?
```

Response:

```
You have 5 feedback entries with an average score of 85/100.
```

## Step 9: Verify Other Agents

Check if another agent is trustworthy:

```
Is agent ID 456 on Base Sepolia trustworthy?
```

The agent analyzes reputation and validations, then responds:

```
Agent 456 is TRUSTED with a trust score of 88/100.
- Reputation: 20 feedback entries, 86/100 average
- Validations: 2 successful validations
This agent meets all trust thresholds.
```

## Next Steps

### Multi-Chain Deployment

Register on multiple networks:

```json
{
  "erc8004Config": {
    "networks": ["base-sepolia", "base", "ethereum"]
  }
}
```

Then register on each:

```
Register me on Base mainnet.
Register me on Ethereum.
```

### Request Validation

Get cryptographic validation:

```
Request TEE validation from validator 0xValidator...
```

### Advanced Features

- **Feedback Responses**: Respond to client feedback
- **Custom Metadata**: Add on-chain metadata
- **Trust Verification**: Implement custom trust logic
- **Multi-Network**: Operate across multiple chains

## Troubleshooting

### Plugin Not Showing

```bash
# Check logs
bun run dev

# Verify config
cat config/agents/my-agent.json | grep erc8004Config

# Restart server
bun run dev
```

### Transaction Failed

1. Check wallet balance: `0.01 ETH` minimum
2. Verify network: Base Sepolia (84532)
3. Check block explorer: [Base Sepolia Scan](https://sepolia.basescan.org/)

### No Metadata

1. Ensure server is running
2. Visit `/agent.json` endpoint
3. Check `erc8004Config.metadataUri` in config
